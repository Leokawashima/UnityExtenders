＜内容＞
・ExColorScope.cs
	Editorで色を変更したい時はGUI.colorとかGUI.backgroundcolorとかを、
	Color.Redとかでいい感じに色を変えてからそのあとに描画するもののためにColor.Whiteとかに戻す処理を行いがち。
	それをUsingステートメントでいい感じにラップしてくれる。
・ExFoldOutHeaderScope.cs
	エディタ拡張を使う人は大体思う。FoldOutHeaderは一回宣言したら、
	必ずHeaderEnd()関数を呼ばなければいけなくてだるい・・・書き忘れがないか心配・・・
	もし関数を呼ぶまでに何らかの理由で処理が中断されたら・・・
	でも安心。Usingsステートメントにすることで描く必要も描き忘れも処理中断が起きた場合も安全に閉じておいてくれる。
	ついでに引数に色を投げれば色も変えてくれる。あら便利。
・ExHeaderGroup.cs
	これはある程度まとまった内容が複数ある場合に真価を発揮するusingステートメント。
	基本的に文字を挿入してそのスタイルを指定できてかつ終わり際に引数で指定していれば、
	その色で区切り線を引いてくれる。勿論なければ引かない。
	これはUsingステートメントでスコープをひくことによってスクリプト上で見出しの下に置きたい変数をまとめやすくなり、
	簡単に言えばスクリプト上でも一見しただけでこの見出しのグループに属した変数なんだなと分かるようになる。
	エディタ拡張を少しいじった人でないとあまり感動しない。
	これなしとこれありの例のスクリプトでも書いて、
	如何にこれがコードを美しくかつ可読性が高いものにしてくれるかがわかるようにしようかと思うくらいには個人的に良いものだと思う。
